IF [V] [conditional] [V]

if V1 [conditional] V2 is true, the code until the following ELSE or ENDIF is run.

In order to do a strong type comparison, the function must be passed the names of variables rather than the value of the variable. I realize this can be kind of a pain since you cannot compare a variable to a value (ie. if a < 10) but meh...

EX: DEFINE INT A 10
EX: DEFINE INT B 20
EX: DEFINE STRING first "A"
EX: IF A < B
EX: //code goes here EX: ENDIF EX: IF A != B
EX: //code goes here EX: ELSE EX: //code goes here EX: ENDIF EX: IF "<&first&>" == B
EX: //code goes here EX: ENDIF
---

ELSE

If and IF conditional evaluates to false... the script will jump to the ELSe (if it exists) and start running code from there.

If you just randomly place an ELSE the script will start scanning for the associated ENDIF to jump to that line.

---

ENDIF

Ends an IF conditional statement. If an IF is true... once it hits the ELSE it will jump to the ENDIF. If the else does not exists and the conditional is false... the script will jump to the ENDIF.
